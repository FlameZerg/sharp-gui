## Why / 为什么

Sharp GUI 当前的图库主要围绕“已生成的 3D 模型作品”组织，用户需要先上传图片并完成推理后，才会在现有侧栏图库中看到条目。用户现在希望把项目部署到本地或局域网后，也能像轻量 NAS 图库一样浏览电脑、移动硬盘、挂载盘或 NAS 目录里的照片：按目录形成相册，以首图作为封面，进入后用美观的瀑布流浏览所有图片，并复用项目现有图片预览能力查看、下载和一键转换为 3D 照片。

这个能力能把 Sharp GUI 从单纯的“生成后查看器”扩展为“本地照片资产入口”：用户先在照片库中挑图，再把单张或多张照片加入 3D 生成队列。由于照片数量可能远大于当前模型作品数量，必须在 UI、缩略图、分页、缓存和跨平台路径处理上先给出明确方案，避免后续实现只在少量图片或单一系统上可用。

## What Changes / 变更内容

- 新增本地照片相册图库：用户可配置一个或多个照片目录，每个目录作为一个相册展示。
- 每个相册使用可用首图或最近图片的缩略图作为封面，并显示名称、图片数量、更新时间和扫描状态。
- 新增照片图库主界面：桌面端使用沉浸式主区域瀑布流，移动端默认多列可调密度，并支持触控捏合切换展示列数。
- 新增相册列表与照片列表 API，支持分页、修改时间/创建时间/名称/大小排序、缩略图 URL、原图预览 URL 和下载 URL。
- 新增照片缩略图缓存机制，列表默认只加载缩略图，不用原图撑列表性能。
- 扩展现有图片预览能力：照片预览必须加载原图而不是缩略图，并支持下载、上一张/下一张、一键加入 3D 生成队列。
- 支持多选照片并批量转换为 3D 照片，转换任务复用现有队列、任务列表和模型图库刷新机制。
- 支持 Windows、Linux、macOS 部署端：目录选择、路径解析、缓存位置、文件名编码和路径遍历防护必须跨平台一致。
- 新增项目风格一致的通用控件：排序下拉、确认弹窗、文本输入弹窗和图库列数滑块均使用自定义玻璃态组件，不使用浏览器原生弹窗/下拉作为照片图库主交互。
- 图库入口放在主应用一级导航，而不是藏在设置中：推荐在侧栏头部下方新增 `模型 / 照片` 分段入口，选择 `照片` 后主区域切换为照片图库工作区。

## Scope / 范围边界

### 包含

- React 主前端中的照片图库视图、相册列表、照片瀑布流、多选工具栏和照片预览增强。
- Flask 后端中的本地照片目录配置、扫描、分页列表、缩略图生成、原图预览、下载和批量加入 3D 生成队列 API。
- `config.json` 中新增照片图库目录配置，并保持旧配置兼容。
- 中英文 i18n 文案、TypeScript 类型、API client、Zustand 状态扩展和关键路径测试/构建验证。
- 跨平台部署端兼容性要求：Windows、Linux、macOS 均可通过手动路径输入使用；本机访问时尽量复用现有系统文件夹选择能力。
- 照片图库相关的项目文档、README 和产品介绍页更新；归档前需要由用户补充统一截图占位图。

### 不包含

- 首版不实现云端相册、用户账号、权限隔离、多用户 ACL 或公网 NAS 鉴权。
- 首版不实现图片编辑、标签 AI 识别、人脸识别、去重、收藏同步或全文搜索。
- 首版不修改 `ml-sharp/` 上游项目，不改变现有 3D 推理算法。
- 首版不改 Legacy `templates/index.html`，除非后续明确要求 legacy 模式也要支持照片图库。
- 首版不引入数据库；元数据索引和缩略图缓存使用文件系统与轻量 JSON 缓存。

## Capabilities

### New Capabilities

- `local-photo-album-gallery`：定义多目录本地照片相册、跨平台目录配置、瀑布流浏览、缩略图缓存、照片预览下载、单张/批量转换为 3D 照片的行为契约。

### Modified Capabilities

- `model-gallery-smoothness`：照片图库应复用同类性能原则，但不改变现有模型图库行为。
- `viewer-reveal-effects-rail`：照片一键转换生成的模型进入现有查看器后，继续沿用现有 3D 查看器与 reveal effects 能力。

## UI 入口方案

推荐入口：在左侧 Sidebar 的上传按钮下方新增一级分段导航：

- `模型`：保持当前任务队列 + 模型作品图库。
- `照片`：显示相册列表、目录管理入口和扫描状态；主内容区切换为照片瀑布流。

原因：

- 这是高频浏览能力，不应放进设置页。
- 照片图库需要完整主区域，不适合塞进当前模型列表侧栏。
- 保留当前上传和模型查看心智：用户在 `模型` 中看 3D 作品，在 `照片` 中选原始照片并发起 3D 生成。
- 移动端可沿用现有侧栏抽屉，打开 `照片` 后主区域成为图库，底部/顶部显示多选操作条，避免小屏拥挤。

## 影响范围

- 后端：`app.py` 新增照片图库配置、扫描、缩略图、原图服务、下载和批量入队 API。
- 前端 API：`frontend/src/api/` 新增照片图库 API 模块，并在 `index.ts` 中导出。
- 前端类型：`frontend/src/types/` 新增照片相册、照片条目、分页响应和转换请求/响应类型。
- 前端状态：`frontend/src/store/useAppStore.ts` 增加当前应用视图、相册选择、多选状态、照片分页状态。
- 前端组件：新增 `frontend/src/components/photoGallery/` 组件组；扩展 `ImageViewer` 或抽象通用图片预览能力。
- 前端样式：新增 CSS Modules，并按 `variables.css` 现有玻璃态、Apple 风格、浅色优先和深色覆盖策略实现。
- 国际化：`frontend/src/i18n/en.json` 与 `frontend/src/i18n/zh.json` 同步新增照片图库相关 key。
- 配置：`config.json` 新增 `photo_gallery_roots`，后端读取时兼容没有该字段的旧配置。
- 测试/验证：至少覆盖后端路径安全、跨平台路径归一化、缩略图生成、前端构建和关键响应式布局。
