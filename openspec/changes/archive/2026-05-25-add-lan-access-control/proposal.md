## Why / 为什么

Sharp GUI 现在已经能把部署端的 `outputs/` 模型目录、`inputs/` 原图和配置好的本地照片目录通过局域网浏览器访问。这个能力让项目更像一个轻量个人 NAS 相册，但当前模型列表、照片相册、缩略图、原图预览、下载、导出等读取链路没有统一门禁；服务又默认绑定 `0.0.0.0:5050`，同一局域网内的设备只要知道地址就能访问。

这带来两个主要风险：

- 隐私泄露：照片原图、相册名称、模型预览、模型文件可以被未授权局域网用户直接查看或下载。
- 误操作风险：部分任务、上传、转换、删除类 API 如果继续对远程开放，可能被同网段用户触发耗时任务或破坏数据。

项目的定位仍是个人本地工具，不适合引入完整账号体系、数据库、OAuth 或复杂 RBAC。更合适的方向是：提供一道可选的轻量局域网门禁；开启后用访问码保护私有资源，同时保留 localhost 作为 owner 管理入口，尽量不打扰日常使用。

## What Changes / 变更内容

- 新增可选局域网门禁能力：开启门禁后，远程设备首次访问私有资源时需要输入访问码，解锁后通过 HttpOnly 会话 Cookie 访问模型、照片、缩略图、原图、下载和导出资源。
- localhost 默认作为 owner 设备，可免登录访问，并继续拥有设置、相册目录管理、重启、删除、批量转换等管理权限。
- 新增认证状态、登录、登出、撤销会话和访问码设置 API，用于前端启动时判断展示应用还是门禁页。
- 新增全局后端访问拦截，门禁开启时统一保护 `/api/*` 中的私有读取 API、`/files/*` 静态工作区资源和照片/模型下载链路；门禁关闭时读取恢复开放，但 owner-only 操作仍限制 localhost。
- 将破坏性或高成本操作明确收紧到 localhost owner，远程已解锁设备默认只拥有浏览、预览、下载权限。
- 增加可选设置 `allow_remote_generation`，默认关闭；只有门禁开启且用户明确开启远程生成后，已解锁远程设备才能提交图片生成或照片转 3D 任务。
- 增加局域网访问安全提示：本机 owner 在门禁未开启或未设置访问码时默认看到提醒；用户可稍后处理或选择不再提示。
- 前端新增轻量解锁体验：远程未授权时显示门禁页；已授权或 localhost 时无感进入现有应用。
- 调整 CORS 策略，避免在开启 Cookie 会话后继续使用宽泛 `Access-Control-Allow-Origin: *` 暴露凭证交互面。

## Scope / 范围边界

### 包含

- Flask 后端认证辅助函数、会话 Cookie、全局请求拦截和受保护端点分级。
- `config.json` 中的轻量访问控制配置，兼容旧配置缺省字段。
- React API client 对 401/403 的识别，启动阶段鉴权状态判断和解锁页。
- 设置页中的局域网门禁配置入口：启用/关闭门禁、设置/修改访问码、会话有效期、撤销所有设备、远程生成开关。
- 中英文 i18n 文案、TypeScript 类型和关键回归验证。
- OpenSpec 规格与任务，用于后续实现和验收。

### 不包含

- 不做多用户账号、角色权限、相册级 ACL、用户组、审计后台或数据库迁移。
- 不做 OAuth、邮件验证码、二维码登录、WebAuthn、LDAP 或公网身份服务。
- 不把访问码放进 URL，不设计分享链接 token，不改变现有导出的离线 HTML 文件内容。
- 不修改 `ml-sharp/` 上游项目。
- 不要求 Legacy `templates/index.html` 获得完整门禁 UI；后端门禁必须保护 Legacy 模式下的私有 API 和文件资源。

## Capabilities

### New Capabilities

- `lan-access-control`：定义局域网访问码、owner 设备、本地免登录、远程会话、私有资源保护、管理操作隔离和远程生成开关的行为契约。

### Modified Capabilities

- `local-photo-album-gallery`：门禁开启时，照片相册列表、缩略图、原图、批量下载和照片转 3D 入口必须经过门禁或 owner 校验；门禁关闭时读取恢复开放，照片转 3D 仍默认 owner-only。
- `model-gallery-smoothness`：门禁开启时，模型列表、缩略图、原图、模型文件、下载和导出必须经过门禁或 owner 校验；门禁关闭时读取恢复开放，删除和批量转换等管理动作仍 owner-only。

## Impact / 影响范围

- 后端：`app.py` 新增认证配置、Cookie 会话、`before_request` 保护、CORS 调整和 endpoint 权限分级。
- 前端 API：`frontend/src/api/client.ts` 需要统一识别未授权状态；新增 `frontend/src/api/auth.ts`。
- 前端类型：`frontend/src/types/` 新增访问控制配置、认证状态和登录响应类型。
- 前端状态：`frontend/src/store/useAppStore.ts` 新增认证状态、owner 状态、门禁配置状态。
- 前端组件：新增门禁页/解锁组件；设置页增加访问控制配置区。
- i18n：`frontend/src/i18n/en.json` 和 `frontend/src/i18n/zh.json` 同步新增门禁相关文案。
- 文档：README 与项目安全说明需说明局域网门禁是可选保护层、启动提醒策略、HTTPS 只解决传输加密，不替代门禁。
